# Meshery Blog Categories

Pick 1-3 categories per post.  The list below comes from a sweep of
existing posts under `collections/_posts/2025/` — stay within it
unless you have a strong reason to introduce a new one.  When in
doubt, prefer the more specific category over `meshery`.

| Category              | Use for                                                                    |
| --------------------- | -------------------------------------------------------------------------- |
| `meshery`             | Default, broad-topic Meshery posts that don't fit elsewhere                |
| `community`           | Community programs, MeshMates, contributor stories, newcomer onboarding    |
| `contributors`        | Contributor recognition, onboarding, governance updates                    |
| `governance`          | Project governance, maintainership changes, CNCF updates                   |
| `releases`            | Release announcements, changelogs, version highlights                      |
| `milestones`          | Project milestones (velocity reports, star counts, anniversaries)          |
| `events`              | Conference talks, webinars, community calls                                |
| `kubecon`             | KubeCon-specific posts (often paired with `events`)                        |
| `security`            | Security posture, advisories, Security Slam updates                        |
| `extensions`          | Meshery extension authoring, marketplace, packages                         |
| `mesheryctl`          | Posts focused on the `mesheryctl` CLI                                      |
| `e2e-testing`         | End-to-end testing initiatives, Playwright, CI signals                     |
| `playwright`          | Playwright-specific tooling posts (often paired with `e2e-testing`)        |
| `multi-tenancy`       | Organizations, RBAC, multi-tenant operation                                |
| `collaboration`       | Cross-project / cross-org collaboration stories                            |
| `platform-engineering`| Platform engineering use cases, internal developer platform stories        |
| `opensource`          | Open-source community / ecosystem posts not tied to a specific feature     |

Notes:

- Existing posts use both hyphenated and unhyphenated forms (`e2e-testing` vs `e2etesting`).  When choosing, prefer the **hyphenated** form for new posts so URLs read naturally; the site does not enforce a canonical spelling.
- Keep the list short.  Three categories is a soft cap; more dilutes the navigation.
- If you genuinely need a new category, add it here and to the post in the same PR.
